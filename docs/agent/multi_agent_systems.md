# 멀티 에이전트 시스템 (Multi-Agent Systems)

단일 에이전트의 한계를 극복하기 위해 전문화된 여러 에이전트가 협업하는 구조입니다.

## 1. 왜 멀티 에이전트인가?
- **복잡도 분산**: 하나의 에이전트가 모든 일을 다 하는 것보다, 일을 쪼개어 전문가에게 맡기는 것이 정확도가 높습니다. (Divide and Conquer)
- **모듈화**: 특정 기능을 수행하는 에이전트를 독립적으로 개발하고 테스트할 수 있습니다.
- **확장성**: 새로운 기능이 필요할 때 새로운 에이전트를 추가하기 쉽습니다.

## 2. 협업 모델의 유형
- **Hierarchical (계층적) 구조**: 'Manager' 에이전트가 전체 계획을 세우고 하위 에이전트들에게 업무를 할당합니다.
- **Sequential (순차적) 구조**: 파이프라인처럼 작업이 A 에이전트에서 B 에이전트로 전달됩니다.
- **Joint Collaboration (공동 협업)**: 중앙 통제 없이 에이전트들이 공유된 캔버스나 대화창에서 자유롭게 의견을 주고받으며 문제를 해결합니다.

## 3. 핵심 메커니즘
- **Agent Hand-off (이관)**: 작업의 소유권을 다른 에이전트에게 넘기는 과정입니다. (예: 상담 에이전트가 결제 관련 질문을 받으면 결제 전문 에이전트에게 넘김)
- **State Sharing**: 에이전트들이 공통으로 알고 있어야 하는 정보를 관리하는 방식입니다.
- **Conflict Resolution**: 에이전트 간의 의견이 충돌할 때 이를 중재하는 로직입니다.

## 4. 멀티 에이전트 설계 시 고려사항
- **Communication Overhead**: 에이전트가 너무 많아지면 서로 대화하는 데 너무 많은 토큰을 소모하게 됩니다.
- **Context Management**: 에이전트 간에 넘겨주는 정보가 너무 많으면 컨텍스트 한계를 초과하거나 중요한 정보가 누락될 수 있습니다.
- **Hallucination Amplification**: 한 에이전트의 잘못된 정보가 다른 에이전트에게 전달되어 오류가 증폭될 위험이 있습니다.
