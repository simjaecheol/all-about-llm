# Ollama

## 아키텍처 및 핵심 기술

### 로컬 배포 중심
- llama.cpp 기반의 경량 서버
- Modelfile 시스템으로 모델 패키징
- 크로스 플랫폼 지원 (macOS, Windows, Linux)

### 사용자 친화성
- 원클릭 모델 다운로드 및 실행
- OpenAI 호환 API 제공
- Docker, Homebrew 지원

## 지원하는 디코딩 전략

### 기본 디코딩 기법
- 모델 양자화 (4-bit, 5-bit GGML/GGUF 형식)
- 기본적인 샘플링 전략
- 멀티모달 모델 지원 (v0.8.0+ 버전)

## 성능 특성

### 로컬 성능
- NVIDIA H100에서 14B 모델 4-bit 양자화 시 75 tokens/s
- 편의성 우선, 대규모 동시 처리에는 제한적
- CPU 및 Apple Silicon GPU 활용

## 장단점

### 장점
- 최고의 사용 편의성
- 로컬 개인정보 보호
- 다중 모델 관리
- 광범위한 플랫폼 지원

### 단점
- 낮은 처리량 (배치 처리 제한)
- 멀티 GPU 설정 비효율성
- 텐서 병렬화 미지원
- 프로덕션 확장성 부족

## 사용 시나리오
- 개인 및 소규모 팀의 로컬 LLM 사용
- 개인정보 보호가 중요한 환경
- 빠른 프로토타이핑 및 실험
- 데스크톱/노트북 환경에서의 개인 AI 도우미
