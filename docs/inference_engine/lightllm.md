# LightLLM

## 핵심 특징

### 경량 설계와 높은 확장성
- 3-process 비동기 협업 (토큰화, 추론, 디토큰화)
- Nopad attention 연산 지원
- 동적 배치 스케줄링

## 최적화 기법

### 고급 메모리 관리
- Dynamic batch 스케줄링
- Token-wise KV 캐시 메모리 관리
- 고성능 라우터와 협업하여 처리량 최적화
- Int8KV Cache (Llama 모델)

### 성능 최적화
- 높은 처리량과 낮은 지연시간
- 효율적인 리소스 활용
- 확장 가능한 아키텍처

## 장단점

### 장점
- 경량화된 설계로 빠른 배포
- 높은 확장성과 처리량
- 효율적인 메모리 사용
- 간단한 설정 및 관리

### 단점
- 상대적으로 새로운 프레임워크
- 제한적인 모델 지원
- 고급 기능 부족

## 사용 시나리오
- 경량 고속 서빙이 필요한 환경
- 빠른 배포 요구사항
- 확장 가능한 아키텍처 필요
- 리소스 효율성이 중요한 환경
