# AX를 위한 AI UX 패턴 점검 체크리스트

[AI UX 패턴 가이드](../ux-patterns.md)를 기반으로, 사용자 인터페이스와 상호작용 방식을 점검하는 체크리스트입니다.

## 1. 업무 맞춤형 UX 패턴 선정
- [ ] 업무 특성과 사용자 자유도를 고려하여 최적의 UX 패턴을 선택했는가? (대화형, 생성형, 에이전트형, 내장형 중)
- [ ] **대화형(Conversational UI)**: 초기 사용자를 위한 추천 질문(프롬프트 가이드)과 긴 응답 대기를 고려한 스트리밍 응답이 적용되었는가?
- [ ] **생성형(Generative UI)**: AI 결과물을 단순 텍스트가 아닌 대시보드나 폼으로 보여주며, 버튼/슬라이더 등 구조화된 피드백 UI가 존재하는가?
- [ ] **자율 에이전트(Agentic UI)**: AI의 작업(클릭, 입력) 과정을 실시간 로그로 보여주며, 즉시 중단할 수 있는 '비상 정지' 버튼이 있는가?
- [ ] **내장형(Embedded/Copilot UI)**: 사용자의 현재 작업 맥락을 자동 파악하고, 워크플로우를 방해하지 않는 인라인/사이드바 형태로 노출되는가?

## 2. 마찰 최소화 (Friction Reduction)
- [ ] 사용자가 프롬프트를 짤 필요 없이 원클릭이나 맥락 인식을 통해 자동으로 다음 작업이 제안되는가?
- [ ] AI의 등장과 보조가 사용자의 본래 업무 흐름을 끊지 않는 매끄러운(Seamless) 구조인가?

## 3. 통제 및 안전망 (Human-in-the-loop)
- [ ] 재무, 보안 등 고위험 작업의 경우 최종 실행 전에 반드시 사용자의 명시적 승인을 거치도록 설계되었는가? (확인 루프)
- [ ] 사용자가 승인 여부를 판단할 수 있도록, AI가 왜 이 결과를 냈는지(Citations, Reasoning) 설명하는 UI가 있는가?
- [ ] 특정 임계값(Threshold)을 넘는 경우에만 인간에게 통보하는 가드레일(예외 기반 확인)이 적용되었는가?
