# RAG (Retrieval-Augmented Generation)

RAG(검색 증강 생성)는 대규모 언어 모델(LLM)의 한계를 보완하기 위해 외부 지식 소스에서 관련 정보를 검색하고, 이를 기반으로 답변을 생성하는 기술입니다. 이 문서는 RAG를 구성하는 핵심 기술 요소들을 소개하고, 각 요소에 대한 심층적인 이해를 돕기 위한 안내서입니다.

## RAG의 핵심 워크플로우

RAG는 크게 **인덱싱(Indexing)**, **검색(Retrieval)**, **생성(Generation)**의 3단계로 구성됩니다.

1.  **인덱싱 (Indexing)**: 외부 문서를 불러와 작고 관리 가능한 단위인 **청크(Chunk)**로 분할합니다. 각 청크는 **임베딩 모델**을 통해 숫자 벡터로 변환되어 **벡터 데이터베이스**에 저장됩니다. 이 과정을 통해 LLM이 문서를 쉽게 검색하고 이해할 수 있는 기반이 마련됩니다.
2.  **검색 (Retrieval)**: 사용자의 질문이 들어오면, 질문 또한 임베딩 모델을 통해 벡터로 변환됩니다. 이 질문 벡터를 사용하여 벡터 데이터베이스 내에서 의미적으로 가장 유사한 문서 청크(벡터)들을 찾습니다.
3.  **생성 (Generation)**: 검색된 문서 청크들이 사용자의 원본 질문과 함께 LLM에 전달됩니다. LLM은 이 추가적인 맥락 정보를 바탕으로 더 정확하고 신뢰성 있는 답변을 생성합니다.

## 1. 임베딩 (Embedding)

임베딩은 RAG의 심장과도 같습니다. 텍스트(질문과 문서)를 의미를 담은 숫자 벡터로 변환하여, 컴퓨터가 단어와 문장의 관계를 이해하고 유사성을 비교할 수 있게 만듭니다.

-   **문서 임베딩**: 문서를 적절한 크기의 청크로 나누고, 각 청크를 벡터로 변환하여 검색 가능한 지식 기반을 구축합니다.
-   **질문 임베딩**: 사용자의 질문을 벡터로 변환하여 문서 벡터와 비교할 수 있도록 준비합니다.

RAG의 성능은 어떤 임베딩 모델을 사용하느냐에 따라 크게 달라집니다. 도메인 특성, 다국어 지원, 성능 등을 고려하여 최적의 모델을 선택해야 합니다.

## 2. 검색 (Retrieval)

검색 단계의 목표는 사용자의 질문 의도와 가장 관련성 높은 정보를 외부 문서에서 정확하게 찾아내는 것입니다. 검색의 품질이 최종 답변의 품질을 결정합니다.

-   **벡터 검색 (Vector Search)**: 임베딩된 질문과 문서 벡터 간의 코사인 유사도를 계산하여 의미적으로 가장 가까운 문서를 찾습니다.
-   **고급 검색 기법**:
    -   **어휘 검색 (Lexical Search)**: BM25와 같은 알고리즘을 사용하여 키워드 일치도를 기반으로 검색합니다.
    -   **하이브리드 검색 (Hybrid Search)**: 벡터 검색과 어휘 검색을 결합하여 의미적 유사성과 키워드 정확성을 모두 잡는 방식입니다.
    -   **재순위화 (Re-ranking)**: 초기 검색 결과의 순위를 더 정교한 모델로 재조정하여 정확도를 높입니다.

- **[검색 기술 상세 가이드](./retrieval.md)**

## 3. 벡터 데이터베이스 (Vector Database)

벡터 데이터베이스는 임베딩된 대규모 벡터 데이터를 효율적으로 저장, 관리, 검색하기 위해 특별히 설계된 시스템입니다. ANN(Approximate Nearest Neighbor) 같은 인덱싱 기술을 사용하여 수백만, 수십억 개의 벡터 중에서도 빠르게 유사도 검색을 수행할 수 있습니다.

-   **주요 기능**: 벡터 저장, 유사도 검색, 인덱싱, 메타데이터 필터링
-   **대표적인 솔루션**:
    -   **오픈소스**: FAISS, Weaviate, Qdrant
    -   **클라우드 서비스**: Pinecone, Chroma, Milvus

- **[벡터 데이터베이스 상세 가이드](./vector_db.md)
