# OpenClaw

OpenClaw는 2025년 말 출시 이후 폭발적인 인기를 끌며 GitHub에서 가장 빠르게 성장한 자율형 AI 에이전트 프로젝트 중 하나입니다. 사용자의 PC, 스마트폰, 비즈니스 환경에서 자율적으로 작업을 수행하는 개인용 AI 어시스턴트 기능을 제공합니다.

## 주요 특징

- **멀티 플랫폼 인터페이스**: WhatsApp, Telegram, Slack, Discord 등 50개 이상의 메시징 앱을 통해 에이전트와 대화하고 명령을 내릴 수 있습니다.
- **방대한 생태계**: 'Claw Hub'를 통해 10,000개 이상의 커뮤니티 스킬(Plugin)이 공유되고 있으며, 이를 통해 웹 브라우징, 코드 작성, 시스템 제어 등이 가능합니다.
- **GitHub 성과**: 2026년 3월 기준 25만 개 이상의 Star를 기록하며 자율형 에이전트 분야의 표준으로 자리 잡았습니다.

## 아키텍처

OpenClaw는 다음과 같은 4개 계층의 논리적 구조를 가집니다.

1.  **Gateway Layer (게이트웨이 계층)**: 중앙 신경망 역할을 하는 WebSocket 서버로, 다양한 메시징 앱의 요청을 오케스트레이션합니다.
2.  **Reasoning Layer (추론 계층)**: LLM과 고도화된 '메가프롬프트'를 결합하여 사용자의 의도를 분석하고 실행 계획을 수립합니다.
3.  **Memory System (메모리 시스템)**: 복잡한 벡터 DB 대신 일반 Markdown 파일을 활용하여 세션 로그와 설정을 관리합니다. 특히 **[메모리 임베딩(Memory Embeddings)](../agent/memory_embeddings.md)** 기법을 사용하여, 로컬 환경에서도 대규모 세션 로그에 대한 시맨틱 검색과 문맥 주입을 지원합니다. 또한 'Write-Ahead Logging' 방식을 차용하여 데이터 안정성을 확보했습니다.
4.  **Skills & Execution Layer (스킬 및 실행 계층)**: 커뮤니티에서 개발된 스킬들을 로드하여 실제 작업을 수행하는 런타임 환경입니다.

## 한계 및 이슈

폭발적인 성장과 함께 'Claw Hub'의 보안 취약점과 악성 플러그인 유포 문제가 제기되기도 했습니다. 이러한 보안 우려를 해결하기 위해 NanoClaw와 같은 보안 특화 대안 프로젝트들이 등장하는 계기가 되었습니다.

### 2026년 보안 취약점 사태

2026년 한 해에만 다수의 치명적인 보안 취약점(CVE-2026-25253, CVE-2026-27001, CVE-2026-30741 등)이 보고되었으며, ClawHub 스토어에서 **1,467개의 악성 스킬**이 발견되는 등 취약한 기본 보안 설정이 문제로 지적되었습니다. 관련 보안 분석은 [동적 에이전트 보안 위협](../security/dynamic-agent-risks.md)에서 다룹니다.

### 경쟁 구도 및 사용자 마이그레이션

2026년 기준, [Hermes Agent](./hermes-agent.md)의 등장으로 자율형 에이전트 시장은 양강 구도로 재편되었습니다. 커뮤니티 여론 분석에 따르면:

- **35%**: 방대한 플러그인 생태계로 인해 OpenClaw에 잔류
- **30%**: 저비용 운영(5일 $10 vs $130)과 영구 기억 장치의 이점으로 Hermes로 이주
- **20%**: OpenClaw를 메인 오케스트레이터로, Hermes를 자가 학습 서브 모듈로 **병행 사용**

Nous Research는 `hermes claw migrate` 명령어를 제공하여, 기존 OpenClaw 사용자의 설정·메모리·스킬·API 키를 1분 내에 Hermes 환경으로 이관할 수 있도록 지원하고 있습니다.
